## Hardware Decision Matrix

| Experiment | Platform | Reason |
|---|---|---|
| Adapter training | **Lab RTX 6000** | 8B model needs 24 GB VRAM for plain bf16 LoRA; run all 4 domains sequentially in one lab session |
| E1 — equivalence on real adapters | **Mac Mini M4** (CPU) | Pure linear algebra on small matrices; no GPU needed |
| E2 — accuracy parity | **Lab RTX 6000** | Long eval pipeline; session limits make Kaggle/Colab risky |
| E3 — timing vs. d_out | **Lab RTX 6000** | Must be CUDA; synthetic only, run in same session as E4 timing |
| E4 equivalence sub-part | **Mac Mini M4** (CPU) | Synthetic + CPU-safe |
| E4 timing sub-part | **Lab RTX 6000** | CUDA; bundle with E3 |
| E5 — β sweep | **Lab RTX 6000** | 7 × E2 eval; too long for Kaggle/Colab |

**Rule of thumb:** Never run code on Mac Mini that needs CUDA. Never run on Kaggle/Colab what might be interrupted mid-session and can't be cheaply restarted.

## How to Use the Agent Prompts

Each `AGENT_PROMPT.md` is a self-contained task description for a coding agent (Claude, GPT-4, Cursor, etc.). To use:

1. **Open a new agent session.**
2. **Feed the agent two files:**
   - `experiments/SHARED_CONTEXT.md` — gives math and code context
   - `experiments/<experiment>/AGENT_PROMPT.md` — gives the specific task
3. **Ask the agent:** `"Implement the script described in AGENT_PROMPT.md. Read SHARED_CONTEXT.md for context."`
4. **The agent writes the script(s).** Review, then follow the corresponding `README.md` to run them.

The prompts are designed to be **agent-agnostic** — no prior session context needed.

## Logging & Results Format

All results land in `results/<experiment_id>/results.json` plus any plot PNGs.

The JSON always has a metadata header:
```json
{
  "experiment": "E1",
  "hardware": "...",
  "dtype": "...",
  ...actual results...
}
```

**Never report a timing number without: hardware, precision, warmup-excluded status, GPU-sync confirmation, seed count.**
